# Project B Documentation

## 2026-05-01 - 加入背景音乐

### 目标

让 `processing.mp3` 成为作品的背景音乐，打开页面时自动开始播放，并循环播放。

### 改动内容

- 在 `sketch.js` 中新增 `bgMusic` 变量。
- 新增 `preload()`，使用 `loadSound("processing.mp3")` 预先加载音频。
- 新增 `startBackgroundMusic()`，统一处理播放逻辑。
- 在 `setup()` 中调用 `startBackgroundMusic()`，让页面打开后尝试自动播放。
- 在 `mousePressed()` 中再次调用 `startBackgroundMusic()`，作为浏览器自动播放限制的兜底。
- 音乐设置为循环播放：`bgMusic.setLoop(true)`。
- 音量设置为 `0.45`，避免盖过视觉体验。

### Debug 记录

- `index.html` 已经引入 `p5.sound.min.js`，所以不需要额外添加音频库。
- `processing.mp3` 已经在 `project-b` 文件夹中，路径可以直接写成 `"processing.mp3"`。
- 现代浏览器通常会阻止“没有用户交互就播放有声音乐”的行为。
- 因此代码会在页面打开时先尝试播放；如果浏览器拦截，用户第一次点击画面时会再次触发播放。

### 结果

- 背景音乐会尽可能在页面打开后自动播放。
- 如果浏览器阻止自动播放，点击画面后音乐会开始，同时原本的 glitch 点击交互也会保留。

## 2026-05-01 - 左右穿梭的水滴形数字流

### 目标

将背景改成左右穿梭的数字流。每组数字流由多行组合在一起，形状接近流线或水滴：中间更长，边缘更短。速度需要同时包含慢速、中速和快速，让背景有远近层次。

### 改动内容

- 新增 `DataFlow` class，专门控制背景数字流。
- 每个 `DataFlow` 包含 5 到 13 条 `lane`。
- 每组数字流只做左右方向移动，不再随机斜向或竖向。
- 使用 `if` 和 `else` 判断数字流从左往右还是从右往左。
- 中间 lane 的长度最长，边缘 lane 的长度更短，形成水滴形轮廓。
- 每条 lane 都会随机选择慢速、中速或快速：
  - 慢速：`0.35` 到 `1.3`
  - 中速：`1.8` 到 `4.2`
  - 快速：`5.0` 到 `9.5`
- 数字字符的选择也改成最基础的 `if` 和 `else`，不使用三元判断符。

### Debug 记录

- 检查了 `sketch.js`，当前没有三元判断符。
- `node --check project-b/sketch.js` 通过，说明语法没有问题。
- 数据流仍然绘制在 `face.display()` 之前，所以它是背景，不会盖住 webcam 人像主体。

### 结果

- 背景数字流变成多行组合的流线形结构。
- 数字流只在左右方向穿梭，更符合这次设定。
- 同一组内的多行有不同速度，画面同时出现快速掠过和慢速漂移。

## 2026-05-01 - 加入麦克风互动

### 目标

加入人和机器之间的声音互动。人的声音不是作为语言被理解，而是作为输入信号被机器读取。声音越明显，机器世界越不稳定。

### 改动内容

- 新增 `mic`，使用 `p5.AudioIn()` 读取麦克风音量。
- 新增 `micLevel`，记录当前麦克风音量。
- 新增 `communicationLevel`，把麦克风音量转换成 0 到 1 之间的互动强度。
- 在 `draw()` 中调用 `updateSoundInput()`，持续更新声音输入。
- 声音越大，背景数字流速度越快。
- 声音越大，背景数字流亮度越强。
- 声音越大，人像 glitch 越强。
- 当声音很大时，会自动触发更强的 `triggerDestabilize()`。

### 代码风格调整

- 移除了 `DataFlow` 中的 object 写法。
- 不再使用类似 `x:`、`y:`、`speed:` 的结构。
- 改成初学者更容易理解的数组：
  - `laneX`
  - `laneY`
  - `laneSpeed`
  - `laneLength`
  - `laneSize`
  - `laneAlpha`
  - `laneCurve`
- 继续保持只使用基础的 `if`、`else`、`for` 和 `while`。

### Debug 记录

- 浏览器可能需要用户允许麦克风权限。
- 如果页面打开时麦克风没有启动，点击画面会再次尝试启动。
- `node --check project-b/sketch.js` 通过。

### 结果

- 人的声音会让机器画面产生反应。
- 作品从单向观看变成了声音输入和机器视觉反馈之间的交流。

## 2026-05-03 - Camera position glitch

### 目标

根据教授建议，尝试让 camera 的位置本身也发生 glitch。这样不只是字符和行错位，整个人像的位置也会突然偏移，像摄像头信号被机器打断。

### 改动内容

- 在 `BinaryFace.display()` 的 `push()` 和 mirror 部分加入 camera position glitch。
- 大部分时间保持正常镜像。
- 少数时候让 `translate()` 使用随机的横向和纵向偏移。
- 偏移概率会受到 `glitchIntensity` 和 `communicationLevel` 影响。
- 声音越明显或 glitch 越强，camera 位置越容易突然错位。

### 结果

- 人像整体会偶尔跳动。
- 声音互动、字符 glitch、背景数据流和 camera 位置错位现在连成同一个机器反应系统。

### Debug 更新

- 之前把随机偏移直接写在 mirror 的 `translate()` 里。
- 因为画面先被 `scale(-1, 1)` 镜像，横向偏移的方向容易看起来只往左。
- 现在改成先保持正常 mirror，再用 `cameraShiftX` 和 `cameraShiftY` 加到绘制位置上。
- `cameraShiftX` 使用左右正负随机，`cameraShiftY` 使用上下正负随机。
- 这样 camera glitch 可以往上、下、左、右不同方向跳动。
- 后来发现偏移幅度太大时，一侧画面会空掉，另一侧会显得被拉出很大一行。
- 因此把横向偏移从画布宽度的 `0.2` 降到 `0.04`，纵向偏移降到 `0.06`。

## 2026-05-06 - 举手触发 HELLO HUMAN

### 目标

加入一个 gesture trigger。观众举起手时，AI 不只是继续重建人脸，而是短暂识别到“这是人类信号”，并用自己的二进制语言回应 `HELLO HUMAN`。

### 概念

在故事里，一千年后人类已经灭亡，只剩 AI 生活在 cyber world 中。观众把自己的脸、声音和身体动作作为信号发送到未来。当 AI 检测到举手这个明确的人类动作时，它尝试回应人类，但这个回应仍然只能由 `0` 和 `1` 组成，并且最后会 glitch 崩散。

### 改动内容

- 新增 `helloX` 和 `helloY` 两个数组，保存 `HELLO HUMAN` 文字的位置。
- 在 `preload()` 中加入 `ml5.handPose()`，使用 hand tracking 检测手部关键点。
- 新增 `hands` 数组保存 handPose 检测结果。
- 新增 `startHandTracking()` 和 `gotHands()`，让模型持续读取 webcam 中的手。
- 新增 `isRaisedHand()`，通过手部关键点的高度判断用户是否举手。
- 当检测到举手时，把 `helloTimer` 设置为 `150`，显示 `HELLO HUMAN`。
- 举手也会调用 `face.triggerDestabilize()`，让 AI 回应同时伴随强 glitch。
- 加入 cooldown，避免举手时每一帧都重复触发动画。

### 动画方式

- `makeHelloText()` 先在隐藏的 `createGraphics()` 画布上写出 `HELLO HUMAN`。
- 程序读取这个隐藏文字的像素，把文字区域转换成一组点。
- `showHelloText()` 再在这些点的位置画 `0` 和 `1`，所以观众看到的不是普通字体，而是 binary characters 组成的字。
- 动画分成三个阶段：
  - 一开始 `0` 和 `1` 从混乱位置聚集。
  - 中间形成清晰的 `HELLO HUMAN`。
  - 最后文字抖动、错位、变透明，像 corrupted signal 一样消失。

### Debug 记录

- `node --check project-b/sketch.js` 通过，说明语法没有问题。
- 保留了原来的 face、DataFlow、声音和 glitch 代码，只在后面追加 gesture trigger 相关函数。
- handPose 需要 webcam 权限。
- 举手检测使用手部关键点的最高点和平均高度，而不是只检测一个点，这样比单点判断更稳定。
- 如果模型还没有加载成功，代码会跳过 hand tracking，不会影响原来的 webcam face、sound 和 glitch。

### 结果

- 观众现在可以通过举手主动向未来 AI 发送身体信号。
- 作品从“AI 看见人脸”变成“AI 识别人类动作并尝试回应”。
- `HELLO HUMAN` 强化了人类和未来机器之间短暂但不稳定的交流。
